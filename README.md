# testArchive

Анализ бинарных файлов .UJRNL внутри архивов .tar.gz
Этот проект предназначен для автоматизированного создания и последующего анализа бинарных файлов формата .UJRNL, которые находятся внутри архивов .tar.gz.
Скрипт извлекает первые байты каждого файла, проверяет их корректность, сравнивает с эталонной сигнатурой и формирует отчёт о состоянии данных.

## Цели проекта
- Автоматически создать большое количество архивов журналов.
- Автоматически обрабатывать большое количество архивов .tar.gz.
- Извлекать из каждого архива файл .UJRNL (если он существует).
- Читать первые 60 байт файла и выводить их в printable‑формате.
- Проверять первые 4 байта как тип файла (magic bytes).
- Сравнивать тип с эталоном, хранящимся в ujrnl_type.sig.
- Создавать эталон автоматически, если он отсутствует.
- Формировать итоговый отчёт и детальный лог.
- Не прерывать работу при ошибках — все ошибки фиксируются.

## Схема стенда
При создании большого количества архивов журналов скрипт циклически подключает USB-флешку в Umka07-2 через реле, ждёт записи журнала (2 минуты), затем отключает питание (2 минуты) и делает следующий цикл.

```
┌──────────────────────┐
│   Блок питания 24 В  │
└───────────┬──────────┘
            │ 24V
            ▼
     ┌────────────────┐
     │      УМКА      │
     │  Питание 24 В  │
     │   USB/Флешка   │
     └───────┬────────┘
             │
             │ Линия к флешке (USB)
             ▼
     ┌──────────────┐
     │   Реле (НО)  │
     │  КОНТАКТЫ    │
     └──────┬───────┘
            │
            ▼
     ┌────────────────┐
     │    Флешка      │
     └────────────────┘

 ┌────────────────────┐
 │    Raspberry Pi    │
 │                    │
 │  GPIO ──► Драйвер ──► Катушка реле
 │                    │
 └────────────────────┘
```

## Анализ содержимого .UJRNL
Скрипт читает первые 60 байт файла и выводит их в консоль в printable‑формате, заменяя непечатные символы точками.

Проверка типа файла. Первые 4 байта используются как сигнатура формата:
- если ujrnl_type.sig отсутствует — создаётся автоматически;
- если существует — каждый файл сравнивается с эталоном;

несовпадения фиксируются в отчёте.

## Устойчивость к ошибкам
Скрипт не падает при:
- битых архивах,
- отсутствии .UJRNL,
- ошибках извлечения,
- слишком коротких файлах,
- любых других исключениях.

Все ошибки отражаются в отчёте.

## Требования
- Raspberry Pi (проверено на Raspberry Pi 4)
- Python 3
- Модуль RPi.GPIO (требуется доступ в Интернет)
- Реле, подключённое к BCM-18 (физический пин 12), включенное состояние по LOW
- Виртуальное окружение

## Подготовка
Создать новую tmux‑сессию (для долгих тестов):
```
tmux new -s testUSB
( [Ctrl + B], затем D -- отсоединиться от сессии, оставив скрипт работать)
(tmux attach -t testUSB -- войти повторно)
```
Создать виртуальное окружение и активировать его:
```
python3 -m venv venv
source venv/bin/activate
(deactivate -- выйти из окружения)
```
Установить зависимости:
```
pip list
pip install RPi.GPIO 
```

## Запуск
```
python testRTC.py
```

## Результаты
Создаётся файл ujrnl_log.txt

Он перезаписывается при каждом запуске и содержит:
- итоговую статистику,
- детальный отчёт по каждому архиву.

В конце работы выводится:
- количество найденных архивов;
- количество успешно открытых;
- количество найденных .UJRNL;
- количество успешно прочитанных;
- количество ошибок;
- совпадения/несовпадения типа;
- случаи, когда тип определить невозможно.

## Настройки
Структура проекта

project/
│
├── data/                 # Папка с .tar.gz архивами
├── ujrnl_type.sig        # Эталонные первые 4 байта (создаётся автоматически)
├── ujrnl_log.txt         # Лог работы (перезаписывается)
└── ujrnl_check.py        # Основной скрипт

## Как работает скрипт
- Проходит по всем файлам в data/.
- Открывает каждый .tar.gz.
- Ищет внутри .UJRNL.
- Читает первые 60 байт.
- Выводит printable‑формат в консоль.
- Проверяет первые 4 байта:
-- создаёт эталон, если его нет;
-- сравнивает с эталоном, если он есть.
- Записывает итог и отчёт в лог‑файл.
- Выводит итог и отчёт в консоль.

Пример вывода в консоль:
```
=== cluster___well___2026_03_12_00_44_46.tar.gz ===
Тип совпал
PRINTABLE: ....cluster___well___2026_03_12_00_44_46

=== cluster___well___2026_03_11_14_28_46.tar.gz ===
Тип НЕ совпал
PRINTABLE: ....cluster_0__well_7451__2026_03_11_14_

==================== ИТОГ ====================
Архивов найдено:            10
Архивов успешно открыто:    10
Файлов .UJRNL найдено:      10
Файлов .UJRNL прочитано:    10
.UJRNL отсутствует:         0
Ошибок при обработке:       0
Тип совпал:                 1
Тип НЕ совпал:              9
Тип не удалось определить:  0
==============================================

Детальный отчёт по каждому архиву:
- cluster___well___2026_03_12_00_44_46.tar.gz: OK
- cluster___well___2026_03_11_14_28_46.tar.gz: Несовпадение типа (первые 4 байта)
```

Где можно использовать
- проверка целостности бинарных журналов;
- анализ формата проприетарных файлов;
- массовая валидация данных;
- подготовка к реверс‑инжинирингу;
- автоматизация QA‑проверок.
